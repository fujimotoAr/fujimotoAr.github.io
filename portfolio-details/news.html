News Sentiment Analysis (Gemastik XV)

Background


For the final stage of Gemastik XV, our team were asked to classify news sentiment whether it's positive, negative, or neutral. There are several features given in the competition dataset, and we were expected to use one or more feature(s) to produce a sentiment label for each test data. Participants were free to use any kind of machine learning model, whether deep learning models or conventional models.

Some information related to this project are as follows:

Objective


This task was aimed to:

  1. To classify news sentiment based on features.

Dataset


The dataset used in this competition could not be disclosed. In short, several data consisting of features related to news entries and their labels were given to the participants. For grading purposes, there was also given the test data similar to the training data without the labels. Validation data could be produced by splitting the training data into training and validation splits.

Data Cleaning


We performed text cleaning for the given dataset according to these aspects:

  1. Converting each text into lowercase form
  2. Removing website URL(s)
  3. Removing punctuations such as the question sign '?' and the asterix '*'
  4. Replacing endline '\n' with a whitespace

Model Architecture


We implemented several architectures for this competition, one of which is the Bidirectional Long Short Term Memory (BiLSTM) architecture.


BiLSTM architecture


Bidirectional Long Short Term Memory (BiLSTM) is a two-way version of the LSTM recurrent neural network architecture. Compared to its single directional version, BiLSTM is capable of capturing contexts from both the beginning and the end of the text, improving its classification performance. Therefore, we elected to use BiLSTM instead of standard LSTM model.

Evaluation and Result


In this project, we use F1-Score as the evaluation metric. After several optimizations, we achieved a F1-Score of 66.79% on test dataset with BiLSTM architecture.
